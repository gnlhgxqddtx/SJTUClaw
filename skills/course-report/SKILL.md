# Course Report Skill

本 skill 用于把用户给出的课程要求转化为一份结构化、可直接提交前再润色的 Markdown 报告草稿。

## 适用场景
- 课程小论文 / 读书报告 / 思政课报告
- 课程学习总结、课程心得
- 课程实验报告

## 工作流程
1. 先确认关键要素：课程 / 主题、字数要求、参考材料位置（如 workspace 中的某个文件）、保存路径。缺失时用合理默认值并在正文中说明。
2. 若用户指明了参考材料文件，先用 `read_file` 读取，把材料要点融入报告，不要凭空编造事实或引用。
3. 依据 `assets/template.md` 的结构组织内容，字数向用户要求靠拢（±10%）。
4. 生成完成后，对照 `references/checklist.md` 自检。
5. 若用户给了保存路径，通过 `create_file`/`overwrite_file`（update tool）写入 workspace，并经过 approval 后保存；未给路径时直接在回答中输出草稿。

## 输出要求
- 使用 Markdown，标题层级清晰，段落连贯，论点有材料支撑。
- 保持学术、克制的语气，不堆砌空话。
- 结尾说明这是草稿，提示用户仍需核对事实与引用后再提交。
